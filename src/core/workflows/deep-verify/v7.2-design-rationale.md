# Deep Verify V7.2 - Context-Aware Verification System

## Philosophy: "Understand First, Verify Second"

**Problem with V7.1:** It assumes specific domains (Security, Logic) and might miss issues in novel contexts (e.g., Creative Writing, UI Design, Educational Content).
**Solution V7.2:** A "Meta-Cognitive" layer that diagnoses the *nature* of the problem before selecting the verification strategy.

**Core Concept:** The **Ontology Scan** (Phase 0). We do not start by looking for bugs. We start by looking for *meaning*.

---

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    PHASE 0: ONTOLOGY SCAN                    │
│           (Method #107: Aristotle's Four Causes)             │
│  • Material: What is this? (Code, Prose, Plan)               │
│  • Formal: What is its structure?                            │
│  • Efficient: What is the context/constraint?                │
│  • Final: What is the user's GOAL?                           │
└──────────────────────────────┬──────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────┐
│                 PHASE 1: STRATEGY SELECTION                  │
│                (Method #141: Method Selection)               │
│  Map "Final Cause" (Goal) to a Verification Strategy         │
│  • Goal: Robustness → Security Strategy                      │
│  • Goal: Innovation → Creative Strategy                      │
│  • Goal: Clarity → Pedagogical Strategy                      │
└──────────────────────────────┬──────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────┐
│                 PHASE 2: EXECUTION (ADAPTIVE)                │
│  Execute the strategy using selected methods.                │
│  (Includes Standard Hygiene Checks implicitly)               │
└─────────────────────────────────────────────────────────────┘
```

## PHASE 0: ONTOLOGY SCAN (#107)

**Instruction:** Analyze the artifact to populate this profile.

| Cause | Question | Insight |
|-------|----------|---------|
| **MATERIAL** | What is the medium? | [Code / Text / Data / Logic] |
| **FORMAL** | What is the structure? | [Class / Essay / List / Graph] |
| **EFFICIENT**| What are the constraints? | [Time / Budget / Legacy / Strict] |
| **FINAL** | **What is the PURPOSE?** | [Security / Speed / Insight / Beauty] |

**Key Output:** The **Primary Purpose** (e.g., "To secure a transaction" or "To teach a concept").

---

## PHASE 1: STRATEGY SELECTION (#141)

Based on the **Primary Purpose** (Final Cause), select a Strategy.

### Strategy A: The Fortress (Robustness)
*   **Trigger:** Purpose is Security, Reliability, Accuracy.
*   **Core Methods:** #109 (Contraposition), #21 (Red Team), #153 (Impossibility).
*   **Focus:** Finding cracks.

### Strategy B: The Muse (Creativity)
*   **Trigger:** Purpose is Idea Generation, Storytelling, Design.
*   **Core Methods:** #41 (SCAMPER), #123 (Newcomb's Paradox), #46 (Genre Mashup).
*   **Focus:** Expanding possibilities.

### Strategy C: The Teacher (Clarity)
*   **Trigger:** Purpose is Explanation, Documentation, Education.
*   **Core Methods:** #72 (5 Whys), #7 (Mentor/Apprentice), #17 (Abstraction Laddering).
*   **Focus:** Reducing ambiguity.

### Strategy D: The Architect (Structure)
*   **Trigger:** Purpose is System Design, Planning, Organization.
*   **Core Methods:** #31 (ADR), #12 (Graph of Thoughts), #68 (Critical Path).
*   **Focus:** Consistency and feasibility.

**Selection:** [Selected Strategy]

---

## PHASE 2: EXECUTION

Run the selected Strategy.

### 2.1 Hygiene Checks (Always Run)
*   #84 Coherence Check (Is it self-consistent?)
*   #83 Completeness Check (Is it finished?)
*   #81 Scope Alignment (Does it match the prompt?)

### 2.2 Strategic Deep Dive
*   Execute the **Core Methods** defined in the selected Strategy.
*   *Instruction:* Apply methods rigorously to the artifact.

### 2.3 Contextual Challenge
*   Use **Method #125 (Simpson's Paradox)**: Does the solution look good in aggregate but fail in specific subgroups/edge cases defined by the **Efficient Cause** (Constraints)?

---

## PHASE 3: OUTPUT

**Report Structure:**
1.  **Ontological Profile:** What we understood this to be.
2.  **Selected Strategy:** How we decided to verify it.
3.  **Findings:**
    *   **Hygiene Issues:** (Universal)
    *   **Strategic Issues:** (Specific to the goal)
4.  **Recommendations:** Tailored to the Final Cause.

---

**Why V7.2 works for Unknown Domains:**
It doesn't look for keywords. It looks for **Purpose**.
*   If the user uploads a "Poem about Rust", V7.1 might trigger "Rust -> Security".
*   V7.2 sees "Final Cause: Beauty/Expression" -> Strategy B (The Muse).
*   It verifies the *Poem*, not the *Rust code*.
